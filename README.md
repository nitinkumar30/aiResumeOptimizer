## 🌟 Overview

This n8n automation workflow takes your existing resume (PDF or DOCX) and a job description, then does the following — fully automatically:

- 📄 **Extracts** structured data from your resume using AI
- 🧠 **Analyzes** the job description for ATS keywords, must-have skills, and priority signals
- ✍️ **Rewrites** your resume content — bullets, summary, skills — to match the JD naturally
- 🖨️ **Compiles** a clean, professional PDF using LaTeX
- 📬 **Delivers** the final PDF directly to your Telegram

No paid APIs. No cloud subscriptions. Just your own n8n instance and a free Google Gemini API key.

## 🗺️ Workflow Architecture

| # | Node | Type | Purpose |
|---|------|------|---------|
| 1 | Resume & JD submitted | Form Trigger | Web form to collect resume file + JD text |
| 2 | Extract from File | Extract From File | Reads raw text from uploaded PDF/DOCX |
| 3 | Resume extractor | AI Agent (Gemini) | Converts resume text → structured JSON |
| 4 | JD extractor | AI Agent (Gemini) | Converts job description → structured JSON |
| 5 | Generated resume unformatted | AI Agent (Gemini) | Fills LaTeX skeleton with ATS-optimized content |
| 6 | Prepare compilation ready | Code (JS) | Cleans LaTeX output, extracts name, builds API body |
| 7 | Compile latex code | HTTP Request | Sends LaTeX to ytotech API, receives PDF binary |
| 8 | Compile & convert to pdf file | Code (JS) | Attaches filename metadata to binary PDF |
| 9 | Send resume via Telegram | Telegram | Sends final PDF as document to your Telegram chat |

## 🔩 Node-by-Node Breakdown

### 1️⃣ Resume & JD submitted — Form Trigger
A built-in n8n web form with two fields:
- **Resume** — file upload (accepts `.pdf` and `.docx`)
- **Job Description** — textarea for pasting the full JD

The form URL is auto-generated by n8n and can be shared with anyone.

---

### 2️⃣ Extract from File
Reads the uploaded resume binary and extracts plain text. Supports both PDF and DOCX formats via n8n's native Extract From File node.

> ⚠️ If your resume is a scanned image-only PDF (no selectable text), extraction will return empty. Use a text-based PDF.

---

### 3️⃣ Resume extractor — AI Agent
**Model:** Google Gemini (via Google PaLM API credential)

Parses the extracted resume text into a detailed JSON object covering:
- Candidate identity, contact, online presence
- Professional summary
- Skill matrix (grouped by category)
- Full work history with projects
- Education, certifications, achievements

Uses strict anti-hallucination rules — only returns what is explicitly present in the document.

---

### 4️⃣ JD extractor — AI Agent
**Model:** Google Gemini

Analyzes the raw job description text and extracts:
- Domain classification and seniority level
- Must-have vs. good-to-have skills
- Technical and non-technical skills
- Responsibilities (core, secondary, strategic, operational)
- Priority keyword ranking (High / Medium / Low)
- Resume optimization suggestions

---

### 5️⃣ Generated resume unformatted — AI Agent
**Model:** Google Gemini

This is the core intelligence node. It receives both JSON objects (from nodes 3 and 4) and fills a complete, locked LaTeX skeleton with:
- Candidate's real content (no hallucination)
- Experience bullets rewritten with strong action verbs
- JD keywords injected naturally throughout
- Skills reordered by JD priority
- ATS-compliant formatting

Outputs a complete, compilable LaTeX document.

---

### 6️⃣ Prepare compilation ready — Code (JavaScript)
Cleans the LLM output before sending to the compiler:
- Strips any markdown fences (` ```latex `) the LLM may have added
- Slices from `\documentclass` to `\end{document}` — discards any stray text
- Extracts the candidate's name from the Resume extractor JSON for the filename
- Sanitizes the name to create a safe filename: `FirstName_LastName_Resume.pdf`
- `JSON.stringify`s the full request body to handle all LaTeX special character escaping safely

---

### 7️⃣ Compile latex code — HTTP Request
**API:** [latex.ytotech.com](https://latex.ytotech.com) (free, no API key required)
**Compiler:** pdflatex

Sends the LaTeX source as JSON and receives a compiled PDF binary in response. Configured to save the response as a binary file field named `pdfData`.

---

### 8️⃣ Compile & convert to pdf file — Code (JavaScript)
Passes the binary PDF through untouched while attaching:
- `filename` — the sanitized `Name_Resume.pdf` string
- `candidateName` — for use in the Telegram caption

---

### 9️⃣ Send resume via Telegram
**Operation:** Send Document

Sends the compiled PDF as a Telegram document with a caption:
```
✅ Resume ready for [Name]
📄 File: Name_Resume.pdf
🤖 ATS-optimized and JD-matched
```

## 📋 Prerequisites

Before importing and running this workflow, make sure you have the following:

- ✅ **n8n** installed (self-hosted) — v2.14 or higher recommended
- ✅ **Google Gemini API key** — free tier is sufficient ([get one here](https://aistudio.google.com/app/apikey))
- ✅ **Telegram Bot** — created via [@BotFather](https://t.me/BotFather)
- ✅ **Telegram Chat ID** — your personal chat ID or group ID where the bot will send files
- ✅ Internet access from your n8n instance to `latex.ytotech.com`

## 🛠️ Setup & Configuration

### Step 1 — Import the Workflow

1. Open your n8n instance
2. Go to **Workflows** → click **Import**
3. Upload the `OptimizeResumeAsPerJD_WORKING.json` file
4. The workflow will appear with all nodes pre-configured

---

### Step 2 — Configure Google Gemini Credential

1. In n8n, go to **Settings** → **Credentials** → **Add Credential**
2. Search for **Google PaLM API** (used for Gemini)
3. Paste your Gemini API key
4. Name it something like `Google Gemini - Resume Optimizer`
5. In the workflow, click each of the three Gemini Chat Model nodes and assign this credential:
   - `Google Gemini Chat Model2` → linked to `Resume extractor`
   - `Google Gemini Chat Model3` → linked to `JD extractor`
   - `Google Gemini Chat Model4` → linked to `Generated resume unformatted`

> 💡 You can use the same credential for all three nodes.

---

### Step 3 — Configure Telegram

#### 3a — Create a Bot
1. Open Telegram → search for **@BotFather**
2. Send `/newbot` and follow the prompts
3. Copy the **Bot Token** you receive

#### 3b — Add Telegram Credential in n8n
1. Go to **Credentials** → **Add** → search **Telegram**
2. Paste your Bot Token
3. Save the credential

#### 3c — Get Your Chat ID
1. Add your bot to a Telegram group, or start a direct chat with it
2. Send any message to the bot
3. Visit: `https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates`
4. Find the `"chat": {"id": ...}` value in the response
5. For a **group**, the ID will be a negative number like `-5273070660`

#### 3d — Update the Telegram Node
1. Click the `Send resume via telegram` node
2. Set your **Chat ID** in the field
3. Assign your Telegram credential

---

### Step 4 — Activate the Workflow

1. Toggle the workflow from **Inactive** to **Active** in the top-right
2. Click the `Resume & JD submitted` Form Trigger node
3. Copy the **Production URL** shown — this is your public form URL

---

## 🚀 Running the Workflow

1. Open the form URL in any browser
2. Upload your resume (`.pdf` or `.docx`)
3. Paste the full job description into the textarea
4. Click **Send ✅**
5. Wait approximately **60–90 seconds** (3 AI calls + compilation)
6. Check your Telegram — the optimized PDF will arrive as a document

> 📱 The form works on mobile too — you can share the URL with others to generate their resumes as well.

## 💡 Enhancement Ideas

These are improvements you can add on top of the current working flow:

### 📧 Email Delivery (Gmail)
Add a **Gmail node** after `Compile & convert to pdf file` to send the PDF as an email attachment. Free with Google OAuth. Great for sharing with recruiters directly.

### ⚡ Parallel AI Processing
Currently `Resume extractor` and `JD extractor` run **sequentially**. They're independent — wire both directly from `Extract from File` to run in **parallel**, cutting total time roughly in half.

### 🔁 Auto-Retry on Compile Failure
Enable **Retry on Fail** (3 retries, 5s wait) on the `Compile latex code` HTTP node. The ytotech free API occasionally has brief downtime.

### 📨 Instant Acknowledgement Email
Add a Gmail/SMTP node right after the Form Trigger to send: *"We've received your resume. Your optimized version will arrive in ~90 seconds."* Makes the experience feel professional.

### 🔔 Error Alert via Telegram
Add an **Error Trigger** workflow that sends you a Telegram message if any node fails, including the error details. Saves you from wondering why nothing arrived.

### 🤖 Model Upgrade
In each Gemini Chat Model node, explicitly set the model to `gemini-1.5-flash` — faster response time and stays well within the free tier for this use case.

### 📊 Google Sheets Logging
Add a **Google Sheets node** at the end to log each submission: candidate name, JD domain detected, timestamp, and ATS score. Useful for tracking usage over time.

### 🌐 Multi-language Support
Add a language detection step after Resume extraction and pass the detected language to the LaTeX generator, enabling resumes to be generated in the candidate's preferred language.

## 🔧 Troubleshooting

| Error | Likely Cause | Fix |
|-------|-------------|-----|
| `Missing \begin{document}` | LLM wrapped output in markdown fences | The `Prepare compilation ready` node handles this automatically — check `latexPreview` in execution output |
| `MISSING_COMPILATION_SPECIFICATION` | Double `==` in HTTP body expression | Ensure body field uses single `=` expression: `={{ $json.requestBody }}` |
| `XeTeXglyph` error | `fontawesome5` package used | Remove `\usepackage{fontawesome5}` from the LaTeX skeleton in `Generated resume unformatted` |
| `join not defined` | Old template+data architecture | Ensure you're on the current architecture — LaTeX goes directly to compiler, no template/data split |
| Empty PDF / blank resume | Resume is a scanned image PDF | Convert to text-based PDF first, or use DOCX format |
| Telegram bot not sending | Bot not added to group | Add the bot to your group and make it an admin |
| Gemini API errors | Rate limit on free tier | Add a **Wait** node (5s) between AI agents, or upgrade to Gemini Pro tier |

## 🧠 How the AI Optimization Works

The workflow uses a **three-stage intelligence pipeline**:

**Stage 1 — Resume Intelligence**
The resume extractor doesn't just copy-paste text. It consolidates multi-role employment histories, maps projects to employers by date, reassembles fragmented two-column layouts, and standardizes all date formats.

**Stage 2 — JD Intelligence**
The JD extractor runs a strict detect-only analysis. It identifies high-priority keywords (marked "Required"/"Must"), classifies the domain, maps responsibility types, and generates a placement strategy — telling the next stage exactly which skills go in which resume section.

**Stage 3 — ATS-Optimized Generation**
Armed with both structured JSONs, the LaTeX generator rewrites experience bullets with action verbs, injects JD keywords at the right density, reorders skills by JD priority, and outputs clean, compilable LaTeX — no hallucination, no invented facts.
